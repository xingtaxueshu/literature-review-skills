# Contributing

Thank you for helping improve this literature review skills collection. Contributions should make the skills clearer, safer, and easier for researchers to use while keeping the researcher's judgment in charge.

## What This Project Accepts

- Practical skill workflows for literature discovery, screening, reading, synthesis, and research planning.
- Clear prompts, checklists, and output formats that help researchers think and decide.
- Improvements to examples, wording, structure, and safety boundaries.
- Corrections that make a skill more accurate, transparent, or operational.

## What This Project Avoids

- Skills that promise to write papers, fabricate citations, or replace researcher judgment.
- Long essays that bury the actual workflow.
- Vague advice without concrete steps, inputs, or outputs.
- Examples that present generated academic claims as verified facts.

## Editing A Skill

When editing an existing skill, preserve its purpose and improve its usability. Keep the description precise, make the workflow action-oriented, and ensure outputs are specific enough for a researcher to inspect, revise, or reject.

Before changing scope, check whether the requested improvement belongs in the current skill or should become a separate skill. Prefer small, reviewable edits over broad rewrites.

## Adding A New Skill

A new skill should solve a distinct research workflow problem. It needs a clear description that says when to use it, a compact operational process, and an output format that supports research decisions rather than replacing them.

Include at least one example when it clarifies expected use. The example should show how the skill supports critical judgment, evidence tracking, and transparent uncertainty.

## Example Quality

Examples should be realistic, bounded, and honest about uncertainty. They should demonstrate useful structure without inventing sources, results, or conclusions. Keep the researcher's interpretation and responsibility visible throughout the example.

## Pull Request Checklist

- [ ] The skill has a clear `description` that says when to use it.
- [ ] The workflow is operational, not a long essay.
- [ ] The output format is concrete enough for users to act on.
- [ ] The skill does not promise to write or fabricate academic work.
- [ ] Examples keep the researcher's judgment in charge.
